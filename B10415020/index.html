Chapter2 Page tables

原理說明

xv6利用page table來管理記憶體的使用

每個page table會有2^20條的PTE，每條PTE又對應到他相對應的page

PTE

每條PTE會有32個位元，前面20個位元用來當作page table的索引(用來搜尋page table)，後面12位元則是分配給每個page table的2^12byte的內存

每條PTE中包含了一些標誌位元

PTE_P:表示PTE是否存在於page table中，若沒有則表示此page不允許被使用

PTE_W:表示是否能對此page做"寫"的功能，若不能則只能做讀取功能

PTE_U:控制使用者程序是否能使用該page，若不能則只有kernel能使用

搜尋page table

首先你會有一個目前正在使用的page table，當你要使用某個記憶體空間時，會使用PTE中的第11至20位元，確認要使用的記憶體空間是存在於目前page table中，若不存在，則再使用第1至10位元轉移至其他page table，取得你所要操作的記憶體空間。

如何使用page table

對於每一個process都會有屬於他們自己的page table，process的user memory從0開始，最多能夠增長到KERNBASE,這使得一個process最多只能使用2GB的memory。當process向xv6要求更多的memory時，xv6首先要找到空閒的physical page，然後把這些page對應的PTE加入該process的page table中，並讓PTE指向對應的physical。

呼叫walkpgdir找到每個virtual addresses對應的physical addresses，並初始化他的flag，再將PTE_P設為有效使他能夠被引用

一開始使用namei打開二進製文件
檢查ELF header的開頭是否符合"magic number"
呼叫setupkvm建立一新的page table
呼叫allocuvm為每個ELF段分配內存，再呼叫loaduvm將ELF段的內容載入內存中
呼叫allocuvm分配兩頁memory
儲存舊的page table
更新process的page table
更新process的空間大小
呼叫switchuvm來切換page table
呼叫freevm來釋放舊的page table
